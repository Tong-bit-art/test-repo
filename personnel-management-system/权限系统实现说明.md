# 人员库管理系统权限系统实现说明

## 一、系统概述

本系统实现了严格隔离的两级权限系统，基于Java Spring Boot + Vue3 + MySQL技术栈，确保数据绝对安全，前后端双重校验。

### 1.1 角色定义
- **管理员（ADMIN）**：系统最高权限，无任何限制
- **录入员（ENTRY）**：权限完全锁死，仅能执行信息录入操作

### 1.2 默认测试账号
- 管理员：admin / 123456
- 录入员：entry / 123456

**文件位置**：`RequireRole.java`

**用途**：标记需要特定角色才能访问的方法或类

**使用示例**：
```java

    return personnelService.findByFilter(filter, getCurrentUser());
}
```

#### 2.4.3 OperationLogController

### 2.5 数据权限隔离

#### 2.5.1 录入员数据隔离
在PersonnelService中实现：
```java
// 录入员只能查看自己录入的人员
if (currentUser != null && currentUser.getRoles() != null && 
    currentUser.getRoles().stream().anyMatch(role -> "ENTRY".equals(role.getName()))) {
    predicates.add(cb.equal(root.get("createdBy").get("id"), currentUser.getId()));
}
```

#### 2.5.2 用户身份验证
每个Controller方法都通过getCurrentUser()获取当前用户：
```java
private User getCurrentUser() {
    Authentication authentication = SecurityContextHolder.getContext().getAuthentication();
    if (authentication == null || !authentication.isAuthenticated()) {
        throw new RuntimeException("用户未登录");
    }
    return ((CustomUserDetails) authentication.getPrincipal()).getUser();
}
```

## 三、前端权限实现

### 3.1 路由权限控制

#### 3.1.1 路由配置
**文件位置**：`router/index.js`

**功能**：
- 基于角色的动态路由
- 路由元信息包含权限要求
- 路由守卫拦截未授权访问

**路由配置示例**：
```javascript
{
  path: '/admin',
  component: AdminLayout,
  meta: { requiresAuth: true, requiresRole: 'ADMIN' },
  children: [
    {
      path: 'personnel',
      component: PersonnelList,
      meta: { title: '人员信息管理' }
    }
  ]
}
```

#### 3.1.2 路由守卫
**功能**：
- 验证用户登录状态
- 验证用户角色权限
- 自动跳转到合适的页面
- 防止越权访问

**守卫逻辑**：
```javascript
router.beforeEach((to, from, next) => {
  // 检查登录状态
  if (!isAuthenticated()) {
    next('/login')
    return
  }

  // 检查角色权限
  if (to.meta.requiresRole) {
    if (!hasRole(to.meta.requiresRole)) {
      // 跳转到合适的页面
      if (hasRole('ADMIN')) {
        next('/admin/personnel')
      } else if (hasRole('ENTRY')) {
        next('/entry/personnel')
      }
      return
    }
  }

  next()
})
```

### 3.2 权限工具函数

**文件位置**：`utils/permission.js`

**提供的函数**：
- `hasPermission(requiredRole)` - 检查是否有指定角色
- `hasAnyPermission(requiredRoles)` - 检查是否有任一指定角色
- `isAdmin()` - 检查是否为管理员
- `isEntry()` - 检查是否为录入员
- `checkPermission(requiredRole)` - 检查权限并提示错误
- `checkAnyPermission(requiredRoles)` - 检查任一权限并提示错误
- `getUserRoles()` - 获取用户角色列表
- `getUsername()` - 获取用户名
- `getUserName()` - 获取用户姓名
- `getUserId()` - 获取用户ID

### 3.3 组件级权限控制

#### 3.3.1 AdminLayout组件
**功能**：
- 根据用户角色动态显示菜单项
- 显示用户角色标签
- 管理员显示所有菜单
- 录入员仅显示"人员录入"菜单

**菜单控制示例**：

```vue
  <span>人员管理</span>
</el-menu-item>

<el-menu-item v-if="isAdmin || isEntry" index="/entry/personnel">
  <el-icon><Plus /></el-icon>
  <span>人员录入</span>
</el-menu-item>
```

#### 3.3.2 EntryLayout组件
**功能**：
- 显示用户角色标签
- 提供退出登录功能
- 简化的布局适合录入员使用

### 3.4 按钮级权限控制

**实现方式**：
- 使用v-if指令结合权限函数

**示例**：
```vue

  删除
```

## 七、部署说明

### 7.1 数据库初始化
1. 执行schema.sql创建数据库和表结构
2. 执行update_password_and_logs.sql更新密码和创建操作日志表
3. 验证默认账号是否创建成功

### 7.2 后端部署
1. 确保Java 17+和Maven已安装
2. 执行`mvn clean package`打包
3. 执行`java -jar target/system-1.0.0.jar`启动服务
4. 确认服务运行在http://localhost:8080

### 7.3 前端部署
1. 确保Node.js和npm已安装
2. 执行`npm install`安装依赖
3. 执行`npm run dev`启动开发服务器
4. 访问http://localhost:3000
